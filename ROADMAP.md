# Roadmap

This roadmap is intended to make contribution areas obvious and to help maintainers prioritize work.

## Near-Term Priorities

- Expand automated testing beyond syntax validation.
- Add more lesson depth for pointers, memory, file I/O, and debugging.
- Improve empty states and onboarding for first-time users.
- Harden execution runner isolation and error handling.
- Add more multilingual lesson bundles beyond the current UI translation scope.

## Good First Contributions

- fix UI copy and documentation gaps
- improve lesson examples and sample programs
- add or refine translations
- improve accessibility labels and keyboard support
- add small visual polish improvements to the workbench
- expand README screenshots, diagrams, and demo assets

## Medium-Term Improvements

- persist user code drafts
- add more structured quizzes or practice checkpoints
- improve profile recommendations
- add clearer per-step diff views for variable changes
- support more lesson packs and educator-authored content

## Stretch Goals

- collaborative teaching or classroom mode
- shareable execution links
- richer AI tutoring with source-linked explanations
- remote runner observability dashboards
- a separate isolated execution service for large-scale deployment

## Areas Where Contributors Can Have High Impact

- Frontend UX and accessibility
- lesson authoring and curriculum quality
- backend reliability and observability
- security hardening around auth and execution
- cloud deployment and infrastructure
- contributor onboarding and documentation
